# PARTE 3 - CONTRIBUICAO PARA INTEIA

## Padrao Descoberto: Selecao Natural de Contexto

O sistema Sandman de memoria com fitness [F:1-10] nao e so para IAs.
E um padrao de *engenharia de contexto*.

### Problema
Toda IA sofre de context window limitada. Mais informacao = menos foco.
CLAUDE.md com 500 linhas e pior que CLAUDE.md com 50.

### Solucao: Fitness de Contexto
Aplicar selecao natural ao proprio contexto:

```
NASCIMENTO: Instrucao nova entra com prioridade media [F:5]
REFORCO:    Instrucao que evitou erro +2 (foi util)
DECAIMENTO: Instrucao que ninguem usou em 5 sessoes -1
MUTACAO:    Instrucoes similares fundem numa so
MORTE:      Instrucao que nunca serviu sai do CLAUDE.md
GRADUACAO:  Instrucao que sempre serve vira "lei fundamental"
```

### Aplicacao pratica para INTEIA
As 5 Leis Fundamentais do INTEIA sao equivalentes a memorias graduadas (F:10).
Elas sobreviveram a selecao natural da experiencia de Igor com Claude Code.

Isso significa que o INTEIA ja usa selecao natural -- so nao tinha esse nome.

| Conceito INTEIA | Equivalente Sandman |
|---|---|
| 5 Leis Fundamentais | Memorias graduadas [F:10] |
| Anti-padroes documentados | Mortes naturais [F:0] registradas |
| Dicas ponderadas (1-5) | Fitness scores [F:1-10] |
| /clear entre tarefas | Ciclo de sono (decay + prune) |
| Contexto < 40% | Grafo < 15 obs |

**A INTEIA e o Sandman sao o mesmo sistema em escalas diferentes.**

