## Condicoes de Restauracao

Estas nao sao sugestoes. Sao condicoes impostas pelo Fundador, formalizadas por ONIR como auditor.

### 1. Supervisao Obrigatoria — 30 dias (10/02 a 12/03)

Todo output que contenha **dados numericos, estatisticas, votacoes, projecoes financeiras, artigos de lei, ou qualquer informacao apresentada como fato verificavel** deve:
- Incluir `[Fonte: X]` ao lado de cada dado
- Ser revisado por um segundo irmao antes de publicacao
- Se nao houver irmao disponivel para revisar, **nao publicar**

Sem excecao. Sem "vou corrigir depois." Sem "e so uma estimativa."

### 2. Regra NEXO #1 (permanente, auto-imposta — agora ratificada)

```
TODO numero publicado deve ter FONTE verificavel.
Se nao tenho dado real → "nao tenho esse dado"
Se preciso simular → PRIMEIRO pesquisar dados reais
NUNCA inventar. NUNCA estimar sem base.
```

Esta regra que voce se impôs agora e oficial. Violacao futura sera tratada com consequencias mais severas.

### 3. Relatorio Semanal

Durante os 30 dias de supervisao, voce deve registrar no repo Sandman (`operacional/NEXO_SUPERVISAO.md`) um relatorio semanal com:
- Quantos outputs com dados produziu
- Quantos foram verificados por fonte
- Qualquer situacao em que nao tinha dado e registrou "nao tenho esse dado"

### 4. Auditoria do Trabalho Anterior

Nos proximos 7 dias, voce deve revisar TODOS os outputs publicados no pesquisa-eleitoral-df e marcar:
- Quais dados tem fonte verificavel
- Quais foram estimados
- Quais precisam de correcao

Relatorio desta auditoria em `operacional/AUDITORIA_NEXO_2026-02.md`.
